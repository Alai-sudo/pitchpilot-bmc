01商业模式画布

基于 Osterwalder 九宫格框架。中间一栏（价值主张）是分析重心；左侧是成本与能力侧，右侧是客户与收入侧。

08 Key Partnerships关键合作
  • Dify / LangGenius：分发渠道 + 共同营销，workflow 节点集成
  • Atlassian (Confluence Marketplace)：唯一以企业知识库为入口的 deck 工具
  • Anthropic：重度 Claude 使用方，争取 startup credit + co-marketing
  • 品牌 / 咨询公司：他们做 brand guideline，PP 做执行；联合销售
  • 企业 SSO / IAM 供应商：Okta / Azure AD 上架是 enterprise 准入门槛
07 Key Activities关键活动
  • 内部 dogfooding 跑稳：Dify BD 全员日用，追踪"准备时间"指标
  • Confluence 集成做成杀手锏：OAuth + 自动同步 + 引用回链
  • 品牌 skill 标准化：5 分钟上传 PDF brand guideline 自动生成 skill
  • PPTX 导出质量打磨到能直接交付客户
  • 3-5 个外部 design partner 试点
06 Key Resources关键资源
  • Arx 平台（Tulpa / Engine / Oasis 抽象）— 真正的工程壁垒
  • pp-writer + ppt-master skill 库— talent 越多越难复制
  • Claude Code session 持久化 + agent-in-sandbox— 工程纵深
  • Dify 内部 BD 网络— 现成的 design partner pool
  • Confluence 解析能力（@mention / 权限继承 / 增量同步）
02 Value Propositions价值主张
  • 核心销售团队每场客户会议的准备时间从 4 小时降到 15 分钟，且 100% 符合品牌规范
  • 差异点 1你的知识库 → 你的 Pitch。唯一一个把 Confluence / Notion 当作一等公民的 deck 生成器
  • 差异点 2品牌合规由 AI 兜底。Skill 系统强制套用 brand guideline，对企业是刚需
  • 差异点 3可扩展的 Agent + Talent 平台。企业可 fork 加自己的行业 / CRM / 合规 talent
  • 非诉求不和 Gamma / Tome 比"AI 生成的精美 deck"——那条赛道已商品化
04 Customer Relationships客户关系
  • Sales-assisted PLG：免费 workspace + 限 deck 数，达阈值后销售跟进
  • Dogfooding story：每场 Dify 客户会议都是 case study
  • 模板 / Talent 社区：用户贡献行业模板，构建网络效应（中期）
  • 企业账户管理：CSM 跟进 seat 扩张和续费

  • P0Dify Marketplace / 插件，零成本分发到 AI-Native 早期用户
  • P0内部 BD 直销，"我们自己用它打单"是最强 reference
  • P1Confluence Marketplace 上架（独有差异化锋利）
  • P1内容营销："AI Native 销售工作流"叙事
  • 不做Product Hunt + Twitter viral——非战场，无武器
01 Customer Segments客户细分
  • L1 已落地Dify / LangGenius 内部 BD、Solutions、合作伙伴
  • L2 主战场中型 SaaS / 咨询公司的 Sales Enablement / Marketing 团队（已用 Confluence / Notion 沉淀知识）
  • L3 不打个人创业者 / 顾问 / PM——红海，Gamma / Tome / Copilot 已饱和
  • 未来金融、政府、医疗：data-never-leaves 的 on-prem 客户
09 Cost Structure成本结构
  • 最大头LLM token：单 deck 约 $1-5（Opus / Sonnet 混用，含 sandbox tool call）。必须靠 prompt caching + Haiku 降级 + workspace BYOK 转嫁。这是生死线。
  • EKS / R2 / D1 基础设施：低，CF 版基本免费档可扛
  • 销售人力：$150-200K / 年 / sales，5-10 个付费客户后再招
  • 工程：当前 1-2 人足够，重心从功能转向集成与质量
  • Skill / 模板内容生产：与品牌 / 内容团队协作分摊
05 Revenue Streams收入来源
  • 主线Team / Enterprise SaaS：$25-50 / seat / mo，10-seat 团队 ≈ $3-6K ARR；年付折扣，Confluence + SSO + 品牌 skill 在 Team 起步
  • 次线Self-host / On-prem License：Arx + EKS 打包，年费 $30-100K（金融 / 政府 / 医疗，数据不能出网）
  • 远期API / Embedded：deck 生成能力开放给 Dify workflow 或第三方 embed，按 usage 计费
  • 不做个人 free-forever——L3 不是客户，会烧光 Anthropic 配额

02可行性判断

两面看：哪些路径必须放弃，哪些路径真实存在。

不可行需要放弃的路径

  1. 个人 PLG 不可行。Gamma 估值 $2.1B、月活 5000 万；Tome 已被 Notion 收购；Microsoft Copilot 在 PowerPoint 里免费送同样能力。没有消费品牌、没有增长团队、没有 viral loop——确定的失败。
  2. "Deck 生成"本身正在被商品化。LLM 进步会让所有人都能生成"还行"的 deck，差异化窗口正在关闭。生成质量不是护城河。
  3. 多 agent + sandbox 的工程复杂度是负担。Gamma 用一个 LLM 调用就能产生体面 deck；PP 用 Arx + Oasis + 多 tulpa。如果不能翻译成用户可见价值（"agent 能调你 CRM 拉真实 lead"），它就是成本。

可行真实存在的机会

  1. B2B Sales Enablement 切入可行。Showpad / Highspot / Seismic 是 $1B+ 市场，但他们都是"管理已有素材"，没人做"AI 自动生成"。空挡真实存在。
  2. Confluence-native 可行且独特。Atlassian 生态里没有像样的 AI deck 工具，且 Confluence 用户天然是企业付费客户。
  3. Dify 内部分销可行。不需要从零冷启动，LangGenius 客户就是 ICP，分发成本极低。

0312 个月路线图

三阶段，每阶段一个清晰的证伪条件。不满足就回到上一阶段重新审视定位，不要硬推。

Phase 1 · 90 天

证伪市场假设

  • 停止追功能，dogfooding 跑稳；Dify 内部 ≥ 10 个 BD 把 PP 当日常工具
  • 选 5 个 Dify 现有企业客户做 design partner（免费试用），条件是接入他们的 Confluence + 准备真实客户会议
  • 一句话定位敲定，主页所有 copy 重写。删掉"Zen workspace / 黄金分割"这些消费向叙事
  • 产出 3 个 case study（"X 公司销售团队会议准备时间从 4h → 20min"）
  • 证伪条件：design partner 续用率 < 60% → 重审 ICP
Phase 2 · 6 个月

第一个付费客户 + 定价验证

  • 推 Team 版 $30 / seat / mo（年付 $25），最小 5 seat 起卖
  • Confluence Marketplace 上架（最大杠杆动作）
  • Brand skill 上传工具上线：5 分钟把 PDF brand book 转成 skill
  • 关闭所有 L3 入口，不接散户
  • 目标：5-10 个付费团队，$50-100K ARR
  • 证伪条件：未达成 → 重审定位或砍项目
Phase 3 · 12 个月

复制 / 自建增长引擎

  • 跑通 sales-assisted PLG 通路：trial → 用量阈值 → 销售跟进
  • 招第一个 AE（必须有 sales enablement / Atlassian 生态背景）
  • 推出 Enterprise self-host 版（Arx + EKS 打包，年费 $50K+）
  • 评估 Arx 单独商业化：PP ARR > $1M 则缓做；触顶则 Arx 可能是更大市场
  • 目标：ARR $500K-1M，3-5 个 enterprise 合同

04最终诚实建议

PitchPilot 工程做得很扎实——多 tulpa、sandbox、skill、双输出、状态机都是真功夫。但它在当前定位下不是一个能独立商业化的产品，它是 Dify 的销售工具 + 一个潜在的 B2B 工具。两条路二选一：

路 A · 保守

作为 Dify 官方组件，不独立商业化。研发投入算 Dify 的 sales enablement 成本。ROI 清晰，天花板低。

路 B · 进取

切到 Confluence-native B2B sales enablement，独立产品。需要 1 个 sales hire + 重写定位 + 砍掉 50% 消费向特性。$10M+ ARR 上限，但需 12-18 个月验证。

不建议第三条路（消费 PLG 单打独斗冲 Gamma）——那是确定的失败。
路 B 的关键决策点在 Phase 1 末：design partner 续用率能否达到 60%。达不到，回到路 A 不丢人。
